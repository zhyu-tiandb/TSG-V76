## 1. 背景与目标

### 1.1 项目背景

TSG 7.x 对外服务引擎基于 **Java** 实现，负责 SSL/TLS 终结、反向代理、认证鉴权、单点登录、NC 模式隧道等核心业务流量。Java 引擎在高并发 SSL 连接、资源占用、协议处理效率及运维复杂度方面存在瓶颈。

OpenResty（Nginx + LuaJIT）具有高并发、低资源占用、原生 SSL/TLS 支持，适合作为网关对外服务引擎。

### 1.2 迁移目标

| 目标 | 说明 | 量化标准 |
|------|------|---------|
| 功能等价 | 所有现有功能在 OpenResty 引擎下 100% 可用 | 0 功能缺失，白皮书附录 A 全量通过 |
| 性能提升 | 提升并发和吞吐能力 | 并发连接数 ≥ Java 版本，SSL 握手延迟降低 ≥30% |
| 行为兼容 | 用户侧、管理侧无感知变化 | 现有配置可无缝导入，管理操作不变 |
| 国密兼容 | 国密 SSL/TLS 协议完整支持 | SM2/SM3/SM4 算法全支持 |
| 部署兼容 | 硬件规格及部署方式不变 | SG-5200/5300/5600/6000 型号支持 |

### 1.3 非目标（Out of Scope）

- 不新增产品功能  
- 不改变客户端交互逻辑  
- 不改变管理页面 UI  
- 不改变硬件平台  
- 不改变部署方式（主路/旁路模式不变）

## 2. 模块迁移分析

### 2.1 迁移分类说明

| 分类 | 标记 | 含义 |
|------|------|------|
| 引擎迁移 | 🔄 | 核心逻辑从 Java → OpenResty/Lua |
| 适配调整 | ⚠️ | 引擎切换后需验证或小幅调整 |
| 无变化 | ➡️ | 管理面/后端/客户端逻辑不变 |

### 2.2 模块迁移矩阵（简化版，可扩展）

| 模块 | 能力 | 当前实现 | 迁移后 | 影响类型 | 迁移要点 |
|------|------|---------|--------|---------|---------|
| 链路安全 | SSL/TLS 加密 | Java SSLContext | OpenResty ngx.ssl / openssl | 🔄 | TLS 协议处理为 OpenResty 原生能力 |
| 链路安全 | 国密 SSL | Java 国密 SDK | OpenResty + 国密 OpenSSL | 🔄 | 验证 Tongsuo/国密 OpenSSL 兼容性 |
| 用户认证 | 数字证书 | Java 证书链验证 | ssl_certificate_by_lua | 🔄 | CRL/OCSP 验证逻辑 Lua 重写 |
| 用户认证 | 用户名+口令 | Java 认证逻辑 | access_by_lua + 后端 API | 🔄 | 认证逻辑迁移，后端存储不变 |
| 用户认证 | 双因子认证 | Java 组合验证 | Lua 组合验证 | 🔄 | 行为保持一致 |
| 权限控制 | RBAC | Java 授权逻辑 | access_by_lua | 🔄 | 角色-应用映射、访问拦截 Lua 重写 |
| 权限控制 | 动态角色匹配 | Java 规则引擎 | Lua 规则 + 后端策略 | 🔄 | 白名单/黑名单逻辑迁移 |
| 应用保护 | 反向代理 B/S | Java HTTP 代理 | OpenResty proxy_pass | 🔄 | 核心性能提升 |
| 应用保护 | NC 模式 | Java 隧道 | OpenResty stream + Lua | 🔄 | 隧道协议兼容客户端 |
| SSO | 表单代填 | Java Form 解析 | Lua body_filter_by_lua | 🔄 | HTML 解析 + 表单注入 Lua 重写 |
| 统一门户 | 页面渲染 | Java 动态页面 | Lua API + 静态资源 | 🔄 | 页面服务方式变更，但内容不变 |
| 高可用 | 双机热备 | Java 状态同步 | OpenResty + 共享状态 | 🔄 | 会话状态同步需重新设计 |
| 日志审计 | 用户访问日志 | Java 日志记录 | log_by_lua | 🔄 | 日志采集点迁移至 Lua |
| 资源管理 | 用户/角色/终端管理 | Java 后端 | 保持不变 | ➡️ | 管理面逻辑不变 |

> ⚠️ 注：以上为示例，详细矩阵可参考 DeepSeek MD 全量表格。

## 3. 性能与非功能指标

### 3.1 性能目标（以 SG-5300 为例）

| 指标 | Java 基线 | OpenResty 目标 |
|------|-----------|----------------|
| 国密最大新建连接数 | 3,000 次/秒 | ≥4,500 次/秒 |
| 国密 SSL 事务速率 | 5,000 次/秒 | ≥8,000 次/秒 |
| 国密最大并发连接数 | ≤30,000 | ≥50,000 |
| 国密加密吞吐量 | 850 Mbps | ≥1,200 Mbps |
| 接入用户数 | ≤10,000 | ≥15,000 |
| SSL 握手 P99 延迟 | 参考 Java | ≤ 70% Java |

### 3.2 非功能指标

- 内存占用 ≤ Java 版本 70%  
- CPU 使用率不高于 Java 版本  
- 双机热备切换时间 ≤ 3 秒  
- 日志格式与 Java 版本完全兼容  
- 客户端无需升级即可访问

## 5. 风险与应对

| 风险 | 等级 | 影响 | 应对措施 |
|------|------|------|----------|
| 国密 OpenSSL 与 OpenResty 集成困难 | 🔴 高 | 国密功能无法迁移 | Phase 0 PoC 验证 |
| SSO 表单代填复杂 | 🟡 中 | 可能导致功能不完整 | 分阶段迁移，优先主流方式 |
| 双机热备状态同步不成熟 | 🟡 中 | 高可用受影响 | 使用 shared dict + 后端同步 |
| Lua 第三方库不成熟 | 🟡 中 | 第三方认证受限 | 评估后端 API 转发方案 |
| 性能未达预期 | 🟢 低 | 迁移收益不明显 | 保留 Java 版本作为回退 |

## 7. 验收标准

### 7.1 功能等价

- 白皮书附录 A 全部功能项验证通过  
- 反向代理模式（B/S+C/S）功能正常  
- NC 模式功能正常  
- SSO 功能（表单代填、证书透传、Cookie 跨域）正常  
- 用户认证全方式正常  
- RBAC + 动态角色匹配正常  
- 终端安全功能正常  
- 双机热备切换 ≤ 3 秒  
- 审计日志内容一致

### 7.2 性能

- 国密最大新建连接数 ≥ Java 基线  
- 国密 SSL 事务速率 ≥ Java 基线  
- 国密最大并发连接数 ≥ Java 基线  
- 加密吞吐量 ≥ Java 基线  
- SSL 握手 P99 延迟 ≤ Java 的 70%

### 7.3 兼容性

- 现有配置文件可导入 7.6 版本  
- 集中管控平台可正常纳管  
- 客户端无需升级即可接入

---

## 8. 一句话结论

> TSG 7.6 本质为“换引擎不换功能”，核心风险集中在国密兼容性与 SSO 表单代填复杂度。建议 Phase 0 先验证国密可行性，再全面启动迁移。
